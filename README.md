### Mobile-First Competitive Advantage

**Market Reality:** 34% of competitors have poor mobile scores (<50). This is a massive opportunity.

#### Mobile Performance Targets

| Metric | Market Average | Target | Advantage |
|--------|----------------|--------|-----------|
| Mobile Score | 61 | 85+ | Top 10% |
| LCP (Mobile) | >3.5s (est.) | <2.5s | Faster than 80% |
| CLS | >0.15 (est.) | <0.1 | Better UX than 70% |
| TTFB | >500ms (est.) | <200ms | Server excellence |

#### Mobile UX Requirements

**1. Click-to-Call Optimization**
```
- Phone number visible in header (always)
- Sticky CTA bar on mobile with phone + "Get Quote" button
- Phone number tappable on all pages
- Track calls with call tracking (CallRail, etc.)
```

**2. Mobile-Specific CTAs**
```
- "Tap to Call" instead of "Call Now"
- Thumb-friendly button sizes (minimum 48px)
- Bottom navigation for key actions
- Swipeable photo galleries
```

**3. Speed Optimizations**
```
- Lazy load images below the fold
- Use WebP image format
- Critical CSS inlined
- Defer non-essential JavaScript
- No render-blocking resources
- Use a CDN (Cloudflare free tier is sufficient)
- Enable Brotli compression
- Cache static assets (1 year)
```

**4. Mobile Content Prioritization**
```
- Phone number and CTA above the fold
- Key info (services, service area) in first scroll
- FAQ sections collapsed by default
- Testimonials in carousel format
- Maps lazy-loaded
```

#### Mobile-First Implementation Checklist

- [ ] Responsive design (not separate mobile site)
- [ ] Touch targets 48px minimum
- [ ] No horizontal scrolling
- [ ] Fonts 16px minimum (no zoom required)
- [ ] Forms optimized (large inputs, proper keyboards)
- [ ] Click-to-call on all phone numbers
- [ ] Sticky header with phone number
- [ ] Bottom CTA bar on service pages
- [ ] Images optimized for mobile bandwidth
- [ ] Video autoplay disabled on mobile
- [ ] Google Maps embed responsive
- [ ] AMP pages for blog posts (optional but beneficial)
